# Codex Rules

本目录是项目内 `Codex` 协作时的默认规则来源，规则内容基于当前项目的 `PLAN.md`、`AGENTS.md` 与对应技术栈官方文档整理。

## 阅读顺序
1. `tech-stack.md`
2. `architecture.md`
3. `react-next-style.md`
4. `typescript-quality.md`
5. `tanstack-query.md`
6. `forms-and-validation.md`
7. `git-commit.md`

## 使用原则
- 规则优先解决“技术边界、架构一致性、代码可读性、可维护性”，而不是只约束格式细节。
- 如果实际需求与规则冲突，先判断是需求变化还是规则过期，再决定改代码还是改规则。
- 涉及技术路线调整时，同步更新 `PLAN.md` 与本目录相关规则。
