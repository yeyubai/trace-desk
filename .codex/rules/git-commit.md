# Git 提交规则

## 基本要求
- 提交信息统一使用中文。
- 每次提交只解决一类问题，不把功能、重构、格式化混成一个提交。
- 提交前自查改动是否与当前目标直接相关，无关变更不要顺手带上。

## 推荐格式
- 使用格式：`类型：简要说明`
- 类型统一从以下集合中选择：
- `新增`
- `修复`
- `重构`
- `优化`
- `文档`
- `测试`
- `构建`
- `配置`

## 示例
- `新增：完成知识库列表页基础布局`
- `修复：处理流式回答中断后的状态恢复`
- `重构：拆分文档导入服务与切块逻辑`
- `文档：补充 Codex 项目规则`
- `配置：统一提交信息使用中文规范`

## 禁止事项
- 不使用纯英文提交信息，例如 `feat: add chat page`。
- 不使用无意义提交信息，例如 `update`、`fix bug`、`改一下`。
- 不把“格式化全部文件”与业务改动放在同一个提交里。
