# Milestones

## v1.0 MVP (Shipped: 2026-03-26)

**Phases completed:** 5 phases, 15 plans, 50 tasks

**Key accomplishments:**

- WorkbenchSnapshot 现在由服务端和客户端共享同一份 zod 契约，导入与聊天回流路径不再裸信 JSON。
- Dashboard 现在在所有子页共享统一状态条，用户一进入工作台就能知道当前模式、依赖 readiness、来源量和会话量。
- 总览页现在同时展示知识库概览和 runtime readiness 面板，运行模式与依赖缺口不再只靠原始状态枚举表达。
- 来源记录现在能明确区分“已导入”和“是否可检索”，PDF 与处理中来源不再被错误归类为失败。
- 导入页现在把来源列表和来源详情明确拆开，用户可以先判断状态，再进入原因解释。
- 导入动作现在会在表单附近即时反馈成功、警告或失败结果，并自动把焦点切到最新来源。
- 聊天链路现在具备显式流式协议、最近会话上下文拼装和引用点击联动，为 grounded answer 主链路打下了第一层稳定基础。
- grounded answer 现在会先检查是否存在可用证据，再决定回答还是明确拒答；检索 query 也开始利用最近会话上下文。
- 聊天交互层现在能明确告诉用户“当前用什么模型档位”和“这轮回答正处于什么状态”，Phase 3 的 grounded chat 闭环已经完成。
- 历史会话现在可以在 Sessions 页查看摘要与内容，并通过 `sessionId` 直接恢复到 Chat 继续追问。
- 反馈现在不再只是当前页内的临时状态，而是会进入可恢复的 snapshot，并能被 Sessions/Chat 两边稳定读取。
- Sessions 页面现在既能回看会话，又能给出 grounded 下一步建议和反馈覆盖视图，真正成为历史会话的业务入口。
- 来源导入现在不再是黑盒：用户可以看到正文抽取策略、正文长度、chunk 预览和风险告警，从而判断为什么这条链接能问、不能问，或者问不到。
- RAG 现在开始脱离 demo 逻辑：切块改由 LangChain.js 驱动，检索会参考来源质量，拒答也会带上更明确的命中/未命中诊断。
- 项目的对外表达已经从“能跑的知识问答 demo”切换成“可解释、可验收的企业级 RAG 工作台”口径。
